# セッション更新確認

このルールはセッション終了時に適用されます。

## 確認タイミング

セッション中に以下があった場合、ドキュメント更新をユーザーに提案:

### 1. 技術的決定の変更

- ライブラリ・フレームワーク・サービスの変更
- アーキテクチャ・デプロイフロー変更
- 設計パターン・コーディング規約変更

### 2. 新ルール・制約の追加

- 開発ワークフロー変更
- Git操作ルール追加
- セキュリティ要件追加

### 3. トラブルと解決策

- 実装中に遭遇した問題と解決方法
- 再発防止のため記録すべき内容

## 更新手順

1. **ユーザーに確認**: 「以下の変更があったため更新が必要と判断しました: [内容]」
2. **承認後に更新**: 自動更新しない、必ず確認を取る
3. **該当ファイルに記録**

## 更新先の選択

| 内容 | 更新先 |
|------|--------|
| よくある間違い | `common-mistakes.md` |
| 開発ルール変更 | `development.md` |
| Git操作ルール | `git-workflow.md` |
| ゲーム仕様変更 | `specifications.md` |
| UI実装パターン | `ui-patterns.md` |
| 技術スタック変更 | `CLAUDE.md` |

## 更新しない内容

以下は一時的な情報のためドキュメントに記録しない:

- 一時的なデバッグ情報
- 特定のPRやブランチに関する情報
- 確定していない仮の決定事項

→ PRコメントや Issue に記録する
