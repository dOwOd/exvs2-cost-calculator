# よくある間違い

これらの間違いは**過去に修正済み**です。新しい実装では繰り返さないでください。

## コスト管理

❌ **間違い**: A機とB機が別々のコストを持つ
✅ **正解**: チームで6000コストを共有（`remainingCost`は1つ）

## EX発動条件

❌ **間違い**: EX発動条件 `remainingCost < minCost`
✅ **正解**: `remainingCost <= minCost`（等号を含む）

## 総耐久値計算

❌ **間違い**: 総耐久 = リスポーン耐久の合計
✅ **正解**: 総耐久 = 初期耐久A + 初期耐久B + リスポーン耐久の合計

## パターン表示

❌ **間違い**: 全16パターンを表示
✅ **正解**: 実際の撃墜数（敗北まで）のみ表示、重複排除

## セオリー

❌ **間違い**: 「セオリー準拠」タブを作成
✅ **正解**: セオリー準拠タブは削除済み

理由: 従来のセオリー（高コスト先落ち）とEX発動条件が相反するため、一概に定義できない

## CI/CDワークフロー

❌ **間違い**: GitHub Actionsワークフローで Node.js バージョンをバラバラに指定する
✅ **正解**: 全ワークフローで同一の Node.js バージョンを使用する

現状: `e2e.yml` は Node 20、`storybook.yml` は Node 24 と不整合がある（#90 で統一予定）

## スコープ判断

❌ **間違い**: コスト計算と無関係なゲームメカニクス（バースト/覚醒システム等）をこのツールに追加する
✅ **正解**: このツールの核は「コスト管理・撃墜順パターン分析」。関連しない機能はガイドページでの知識提供に留める

理由: スコープの拡大はUIの複雑化を招き、ツールの本来の価値を損なう（#12, #13 クローズ済み）
