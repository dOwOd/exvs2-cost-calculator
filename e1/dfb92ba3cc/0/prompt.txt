# Issue #181: スマホでの耐久値選択時の機体名表示を改善

## Context

耐久値選択ドロップダウンで、PCではホバーで機体名ポップアップが表示されるが、スマホではホバー操作がないためタップ時に一瞬表示されて消えてしまう。スマホでも耐久値と機体の対応を確認しやすくする。

## アプローチ: ドロップダウン項目に機体名をインライン表示

各耐久値オプションの下に、小さなフォントで機体名をコンパクト表示する。

```
┌───────────────────────────┐
│ 720                       │
│ ゴッドガンダム、マスター… 他10機 │  ← text-xs, truncate
├───────────────────────────┤
│ 680                       │
│ νガンダム、Hi-νガンダム 他22機  │
└───────────────────────────┘
```

- 既存の `formatMobileSuitNames(cost, health)` をそのまま活用（最大2機体名 + 残り数）
- PCのホバーポップアップ（全機体名表示）はそのまま維持

## 変更ファイル

### 1. `src/components/HealthSelector.tsx`

- `formatMobileSuitNames` を `mobileSuitsData` からインポート追加
- `<li>` の内容を2行構成に変更:
  - 1行目: 耐久値（既存）
  - 2行目: `formatMobileSuitNames(cost, health)` を `text-xs truncate` で表示
- 選択状態（`bg-blue-600`）では `text-blue-100` で視認性確保
- `flex items-center` → 縦並びレイアウトに変更（2行表示のため）

### 2. `src/components/__tests__/HealthSelector.test.tsx`（新規）

テストケース:
1. 各耐久値オプションに機体名がインライン表示される
2. 残り機体数が「他X機」形式で表示される
3. 1機のみの耐久値では「他」表示がない
4. 選択状態でも機体名が表示される
5. ドロップダウンの開閉が正常に動作する

## 変更しないファイル

- `HealthDropdownPopup.tsx` - PC向けホバーポップアップはそのまま維持

## 検証

```bash
pnpm test                    # ユニットテスト

pnpm build && pnpm test:e2e  # E2E（既存テストが壊れないことを確認）
```
